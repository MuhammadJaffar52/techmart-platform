One Recommendation
I recommend one small improvement to the roadmap to make it even closer to how professional teams work:
After we finish the 3-node KIND cluster, we should pause Kubernetes and move directly to building the TechMart application (Phase 1). Then we'll containerize it with Docker and Docker Compose. Only after we have a working application will we return to Kubernetes.
That way, every Kubernetes object (Pod, Deployment, Service, Ingress, etc.) will be applied to a real application instead of sample nginx manifests. This mirrors real-world DevOps work, where the platform exists to run applications rather than being learned in isolation.
